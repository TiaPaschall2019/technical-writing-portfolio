# Glossary of Markdown & GitHub Terms

**Markdown**
A simple formatting language that uses plain-text symbols to create headings, lists, links, and other formatting.

**Repository (repo)**
A project folder hosted on GitHub that stores files and tracks changes made to them over time.

**README**
A file, usually written in Markdown, that introduces a repository and explains what it contains.

**Commit**
A saved change to a file in repository, along with a short message describing what was changed. 

**Preview**
A rendered view of a Markdown file showing how the formatting will actually appear, rather than the plain-text symbols.
