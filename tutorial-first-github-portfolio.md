# Tutorial: Creating Your First GitHub Portfolio Repository

## Introduction
This tutorial walks you through creating a GitHub repository to host a writing portfolio. By the end, you'll have a live repository with a README and a sample file, ready to share with others. 

## Prerequisites
- A free GitHub account
- At least one file you'd like to add (such as a .md document)

## Step 1: Create a New Respository
Log in to GitHub and click the "+" icon in the top-right corner, then select "New repository." Give it a clear, descriptive name - this will be the project folder that holds all your files. 

## Step: 2 Add a README
When creating your repository, check the box to include a README file. This becomes the homepage of your repository and is usually the first thing visitors see.

## Step 3: Upload Your First File
Click "Add file," then "Upload files." Drag your document into the browser window, add a short commit message describing what you added, and click "Commit changes"

## Step 4: Link the File in Your README
Edit your README and add a Markdown link pointing to your uploaded file, so visitors can find and open it easily.

## Conclusion
You now have a live respository with a working README and a sample document. From here, you can continue adding more samples to build out a full portfolio.
